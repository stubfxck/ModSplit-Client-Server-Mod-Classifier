# ModSplit (Mod Side Classifier)

Инструмент для авторов Minecraft-сборок, который автоматически определяет, какие моды нужны на клиенте, какие — только на сервере, а какие игрокам ставить не обязательно.

Заебался вручную лазить по страницам каждого мода на Modrinth/CurseForge, чтобы понять client-only он, server-only или опционален? ModSplit делает это одной кнопкой.

## Что делает

- Сканирует папку `mods` и определяет side каждого `.jar`
- Проверяет [Modrinth API](https://docs.modrinth.com/api/) по SHA1-хешу файла
- Если не нашёл — пробует [CurseForge API](https://docs.curseforge.com/) по murmur2-хешу (нужен бесплатный API-ключ)
- Если не нашёл нигде — читает метаданные прямо из jar (`fabric.mod.json` / `mods.toml`)
- Делит моды на 4 группы:
  - обязательны на клиенте
  - есть на сервере, но клиенту не обязательны
  - только серверные
  - неясно (нет данных ни в одном источнике)
- Сохраняет читаемый отчёт в `mod_sort_result.txt`

## Для кого

Создатели модпаков, владельцы серверов, авторы сборок для друзей/комьюнити — все, кто формирует отдельные client-pack и server-pack архивы и не хочет разбирать это руками.

## Установка

Нужен Python 3.9+.

```bash
git clone https://github.com/<твой-юзернейм>/modsplit.git
cd modsplit
pip install -r requirements.txt
```

Зависимость одна — `requests`. Если её нет, скрипт попробует установить её сам при первом запуске.

## Использование

Запусти без аргументов — откроется окно выбора папки с модами:

```bash
python mod_sorter.py
```

Или укажи путь к папке сразу:

```bash
python mod_sorter.py "C:\путь\к\mods"
```

Результат:
- сводка в консоли
- подробный отчёт в `mod_sort_result.txt` рядом с папкой mods

## CurseForge (опционально)

Modrinth работает без ключа. Для подключения CurseForge как fallback-источника:

1. Получи бесплатный API-ключ на [console.curseforge.com](https://console.curseforge.com/)
2. Впиши его в начало `mod_sorter.py`:
   ```python
   CURSEFORGE_API_KEY = "твой_ключ"
   ```

Без ключа этот шаг просто пропускается, ошибок не будет.

⚠️ CurseForge API не отдаёт точный client/server статус, как Modrinth — только подтверждает, что мод там есть. Такие моды попадают в категорию «неясно».

## Ограничения

- Точность зависит от того, насколько корректно сам автор мода заполнил `environment`/`side` или указал client/server-side на Modrinth/CurseForge
- Моды, которых нет ни на одной площадке и без метаданных в jar, классифицировать невозможно физически — это не баг инструмента, а отсутствие данных
